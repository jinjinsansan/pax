# pax 開発記録 — 2026-07-19（初日・1日で監視システム本稼働まで）

## 発端

投資案件「AGL」が「Uniswap上でUSDT→PAXG→XAUT→USDTのアービトラージで、元本$300万に対して日利2%（1日4〜8回の取引）」と主張。GPTとの検証（`ありがとうございます。3枚とも確認しました。.txt`）で、提示された証拠（Etherscanスクショ＋運営出力CSV）ではオンチェーン実行の証明にならないと判明。Tx Hashの開示を求めたが**拒否された**。

そこで「PAXG/XAUT裁定が実際にどの頻度・どの金額・どの純利益で成立するか」をオンチェーンQuoteで継続測定するシステムを構築することにした。仕様書は `PAXG_XAUT_Arbitrage_Monitor_Design_Spec_v1.1.md`。

## AGL検証の結論（開発前に確定）

公開データだけで主張は反証された：

- BingXのPAXG/XAUT板は極薄（日次出来高$76k/$163k、PAXG板上位5本$1,500分）→ CEX執行は不可能
- オンチェーン（Uniswap）のPAXG/XAUT関連プール全体の日次出来高は**約$145万**
- AGLの主張（$300万×4〜8回転/日）には**日$1,200万〜2,400万の出来高**が必要 = 実在する市場の8〜16倍
- 1回あたり必要な純エッジ0.25〜0.5%は、メインプール手数料0.01%の市場構造上ほぼ発生しない
- 固定日利という payout 構造自体がポンジの典型

**判定: ほぼ確実にポンジ・スキーム。** 本システムの30日実測は、この判定の定量的なダメ押し証拠となる。

## 初の実測結果（block 25566524、36 Quote全成功）

「今この瞬間、実際に交換したらいくら戻るか」：

- **全ルート・全金額でマイナス**（ガス控除前の段階で）
- 最良: F (USDT→XAUT→USDT) $1,000 → **-0.12%**
- AGLの主張ルートA (USDT→PAXG→XAUT→USDT) は $10,000 → **$4,369しか戻らない（-56%）**
  - 理由: USDT/PAXG直接プールがダスト。AGLのCSVの経路は流動性的に実行不能と実測で確定
- USDC系ルートC/Dが健全（D $10k: -1.34%）。PAXGの主流動性はPAXG/WETH($13M)
- WETH中継ルートG/Hを追加したが $1kで-3.9〜-7.1%（プール選択ヒューリスティックに課題、TODO）

## 本日踏んだ罠（再発防止）

1. **viemはEIP-55チェックサム検証で大文字アドレスを拒否** → 全アドレス小文字正規化で解決
2. **@types/nodeは使用パッケージごとに明示必須** → ローカルはpnpm hoistingで偶然通るがDockerで失敗
3. **XServer VPSはパケットフィルターが既定で全遮断** → パネルでSSH許可が必要だった
4. **Supabase Management APIはUser-Agent必須**（PythonデフォルトUAだと403 error 1010）
5. viemのWS購読は稀にundefinedブロックを渡す → ガード必須（初回スモークでクラッシュ）
6. ESLint no-irregular-whitespace が通知テンプレートの全角スペースを弾く → skipStrings設定
